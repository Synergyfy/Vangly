# COMPLETE LOCATION ADMIN DASHBOARD REDESIGN PROMPT

Design and redesign the complete Location Admin Dashboard for our mobile-first outreach and engagement platform.

IMPORTANT:
This platform is MOBILE-FIRST.

Most users will use:
- mobile phones
- small screens
- quick actions
- simple workflows

The UI/UX must prioritize:
- mobile responsiveness first
- lightweight navigation
- simplicity
- fast actions
- clean layouts
- outreach-focused workflows
- easy usage for non-technical users

The platform is designed primarily for:
- churches
- ministries
- NGOs
- outreach organizations
- volunteer communities

The platform is NOT:
- enterprise CRM
- project management software
- social media platform

Keep the experience simple, modern, and operational.

---

# 🔷 LOCATION ADMIN ROLE

Examples:
- Branch Admin
- Branch Pastor
- Branch Coordinator
- Location Leader

IMPORTANT:
Location admins ONLY manage their assigned location.

They should NOT:
- manage organization-wide settings
- manage other locations
- access organization wallet
- access global branding settings

Their dashboard is focused ONLY on:
- location performance
- internal teams
- forms
- outreach activities
- operational management

---

# 🔷 LOCATION DASHBOARD STRUCTURE

The Location Dashboard should contain ONLY:

1. Performance
2. Teams
3. Settings

ONLY 3 tabs.

VERY IMPORTANT.

Keep navigation simple and mobile-first.

DO NOT add:
- separate members tab
- separate forms tab
- separate contacts tab
- separate users tab
- separate analytics page

Forms and members should exist contextually inside Teams.

---

# 🔷 MOBILE-FIRST NAVIGATION

IMPORTANT:
This dashboard is primarily used on phones.

DO NOT use:
- heavy sidebars
- enterprise admin layouts
- overcrowded menus
- complex desktop-first navigation

Use:
- top tabs
- bottom navigation
- card-based layouts
- progressive drill-down flow
- stacked mobile sections

Navigation flow should feel:
- fast
- simple
- modern
- touch-friendly

---

# 🔷 LOCATION FLOW

Performance
→ Teams

→ Members / Forms

This flow is VERY important.

Keep it clean and contextual.

# IMPORTANT TEAM PRINCIPLE

Teams are ONLY for INTERNAL USERS.

Examples:
- Workers
- Evangelism Team
- Staff
- Follow-Up Team
- Choir
- Media Team
- Leaders
- Volunteers

IMPORTANT:
External contacts or first timers are NOT teams.

External people:
- do NOT create accounts
- do NOT belong to teams
- do NOT log into dashboards

They ONLY:
- fill forms
- submit responses
- become captured contacts/responses

Keep this product structure clean.
